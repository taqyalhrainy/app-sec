Security Features Implemented

🔐 Authentication & Authorization

  • JWT token-based authentication
  • bcrypt password hashing
  • Role-based access control
  • 24-hour token expiration

🛡️ Input Validation & Sanitization

  • Email format validation
  • Password strength requirements
  • Input sanitization (XSS prevention)
  • Phone number validation

🔒 Data Protection

  • AES-256 encryption for sensitive fields
  • HTTPS headers via Helmet.js
  • CORS with restricted origins
  • Secure environment variables

⚔️ Attack Prevention

  • Rate limiting on login/signup
  • Audit logging for security events
  • Error handling without info disclosure
  • User enumeration prevention

Technology Stack

Backend

  • Node.js + Express.js
  • SQLite Database
  • JWT Authentication
  • bcrypt Password Hashing

Frontend

  • HTML5
  • CSS3
  • Vanilla JavaScript
  • localStorage for tokens

Security Libraries

  • helmet.js (HTTP headers)
  • cors (Cross-origin requests)
  • express-rate-limit (Brute force)
  • validator (Input validation)

Security Tools

  • GitHub CodeQL
  • Snyk vulnerability scanning
  • STRIDE threat modeling
  • DREAD risk assessment

About This Project

This project is designed for an Application Security course to demonstrate practical implementation of security best practices. It covers authentication, authorization, input validation, data encryption, and attack prevention techniques.

The application uses a simple but secure architecture that emphasizes security over advanced features, making it ideal for educational purposes and learning security fundamentals.
